## 生成链路

```mermaid
%%{init: {"flowchart": {"useMaxWidth": true, "htmlLabels": false}}}%%
flowchart TB
    subgraph Request["① 受理请求"]
        direction LR
        PROMPT["输入提示词"] --> TASK["创建异步任务"]
        TASK --> ACCEPT["返回 202 与任务 ID"]
        ACCEPT --> SUBSCRIBE["订阅 SSE"]
    end

    subgraph Generate["② 后台生成"]
        direction LR
        PARSE["解析创作约束"] --> SCORE["生成 Score"]
        SCORE --> VALIDATE["时值与指法校验"]
        VALIDATE --> SAVE["保存版本和导出物"]
    end

    subgraph Result["③ 返回结果"]
        direction LR
        EVENT["发布完成事件"] --> PUSH["SSE 推送"]
        PUSH --> PREVIEW["展示预览和导出入口"]
    end

    SUBSCRIBE --> PARSE
    SAVE --> EVENT
```

第一阶段只负责受理请求并建立进度通道；解析、生成、校验和保存由后台任务完成。

## 核心边界

- Domain 保持纯 Java，不依赖 Spring、HTTP、MyBatis 或 LangChain4j。
- Agent 理解意图，但不直接写数据库或最终 MusicXML。
- `Score` 是生成、修改、校验和导出的唯一事实来源。
- Application 调用端口，Infrastructure 实现端口。
- 当前仍是单轨规则生成；多轨、和声与结构化 SongPlan 是下一阶段。
