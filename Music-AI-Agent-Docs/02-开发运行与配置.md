# 开发运行与配置

导航：[[00-首页]] · [[03-测试与验收]] · [[PROJECT_STRUCTURE]]

## 运行入口

| 方式 | 说明 |
|---|---|
| IDEA | 使用 `.run/MusicAiAgentApplication.run.xml`，默认 H2 和规则解析器 |
| 通用 LLM | 激活 `llm` Profile，提供商、地址、模型名和 Key 从环境变量读取 |
| Docker | MySQL、API、前端、音频服务统一编排 |

选择建议：只改领域规则时使用默认 IDEA 配置；验证真实自然语言解析时启用通用 LLM；联调前后端、数据库和音频服务时使用 Docker。外部开发者使用自己的模型时，优先通过 MCP 结构化约束工具接入，Key 无需进入本服务器。

```text
前端  localhost:5173
API   localhost:8080
MySQL localhost:3306
音频  localhost:8000
```

```powershell
docker compose up -d --build
docker compose ps
docker compose logs api --tail 100
```

启动后应先确认 API 和依赖服务健康，再进行生成请求。SSE 长连接问题优先检查反向代理是否关闭响应缓冲。

## 环境变量

| 变量 | 用途 | 敏感 |
|---|---|---:|
| `LLM_PROVIDER` | 模型提供商：`deepseek`、`openai`或`openai-compatible` | 否 |
| `LLM_BASE_URL` | OpenAI兼容模型服务地址 | 否 |
| `LLM_MODEL` | 模型名称 | 否 |
| `LLM_API_KEY` | 当前部署使用的模型密钥 | 是 |
| `MUSIC_AI_ACCESS_KEY` | 网页/API 登录 | 是 |
| `MUSIC_AI_DB_PASSWORD` | MySQL 业务密码 | 是 |
| `MYSQL_ROOT_PASSWORD` | MySQL root 密码 | 是 |
| `MUSIC_AI_MCP_TOKEN` | MCP Bearer Token | 是 |
| `MUSIC_AI_DB_URL` | JDBC URL | 否 |

启用服务器模型示例：

```powershell
$env:LLM_PROVIDER = "deepseek"
$env:LLM_BASE_URL = "https://api.deepseek.com"
$env:LLM_MODEL = "deepseek-v4-flash"
$env:LLM_API_KEY = "本机密钥"
$env:SPRING_PROFILES_ACTIVE = "llm"
cd music-backend
.\mvnw.cmd spring-boot:run
```

其他OpenAI兼容服务只需替换四个`LLM_*`变量。MCP客户端若调用`generate_guitar_riff_from_constraints`，后端不会调用上述模型。

## 启用 MCP

MCP 默认关闭。启用时必须同时配置一个非空 Bearer Token：

```dotenv
MUSIC_AI_MCP_ENABLED=true
MUSIC_AI_MCP_TOKEN=请在本机生成足够长的随机值
```

Docker Compose 会读取项目根目录的 `.env`。直接在 IDEA 运行 Spring Boot 时，`.env` 不会被 Spring 自动加载，需要在运行配置中加入同名环境变量。MCP 客户端连接 `/mcp` 时携带 `Authorization: Bearer <token>`。

网页工作台仍通过 REST/SSE 使用后端，不会自动经过 MCP。MCP 是给外部 AI 客户端或另一个 Agent 使用的协议入口；两类入口最终复用同一个 `MusicProjectService`。

> [!warning]
> 真实密钥只保存在未提交的 `.env` 或环境变量中，不能进入笔记、日志和截图。

## MySQL 初始化

- `docker/mysql/init.sql` 只在数据目录为空时自动执行。
- 已有数据库应使用迁移更新结构。
- `docker compose down -v` 会永久删除 MySQL 数据卷。
- Navicat 默认连接 `localhost:3306`，用户 `music_ai`，数据库 `music_ai`，密码以本机 `.env` 为准。

## 常见问题

| 现象 | 优先检查 |
|---|---|
| 模型调用失败 | `llm` Profile、Provider、Base URL、模型名、Key与账户是否可用 |
| 页面能打开但生成无进度 | API 是否运行、SSE 请求是否保持连接、Nginx 是否缓冲响应 |
| MySQL 表未创建 | 数据卷是否已有旧数据、初始化脚本是否只在首次启动执行 |
| MIDI/MusicXML 不可下载 | 任务是否完成、导出目录是否可写、文件元数据是否已保存 |
| 默认启动仍要求模型 Key | 是否误启用了 `llm` Profile 或覆盖了默认配置 |

排查时可以记录错误类型、时间和任务 ID，但不得复制密钥、Cookie 或完整认证请求头。
