# 开发运行与配置

导航：[[00-首页]] · [[03-测试与验收]] · [[PROJECT_STRUCTURE]]

## 运行入口

| 方式 | 说明 |
|---|---|
| IDEA | 使用 `.run/MusicAiAgentApplication.run.xml`，默认 H2 和规则解析器 |
| DeepSeek | 激活 `deepseek` Profile，Key 从环境变量读取 |
| Docker | MySQL、API、前端、音频服务统一编排 |

选择建议：只改领域规则时使用默认 IDEA 配置；验证真实自然语言解析时启用 DeepSeek；联调前后端、数据库和音频服务时使用 Docker。

```text
前端  localhost:5173
API   localhost:8080
MySQL localhost:3306
音频  localhost:8000
```

```powershell
docker compose up -d --build
docker compose ps
docker compose logs api --tail 100
```

启动后应先确认 API 和依赖服务健康，再进行生成请求。SSE 长连接问题优先检查反向代理是否关闭响应缓冲。

## 环境变量

| 变量 | 用途 | 敏感 |
|---|---|---:|
| `DEEPSEEK_API_KEY` | DeepSeek 调用 | 是 |
| `MUSIC_AI_ACCESS_KEY` | 网页/API 登录 | 是 |
| `MUSIC_AI_DB_PASSWORD` | MySQL 业务密码 | 是 |
| `MYSQL_ROOT_PASSWORD` | MySQL root 密码 | 是 |
| `MUSIC_AI_MCP_TOKEN` | MCP Bearer Token | 是 |
| `MUSIC_AI_DB_URL` | JDBC URL | 否 |

> [!warning]
> 真实密钥只保存在未提交的 `.env` 或环境变量中，不能进入笔记、日志和截图。

## MySQL 初始化

- `docker/mysql/init.sql` 只在数据目录为空时自动执行。
- 已有数据库应使用迁移更新结构。
- `docker compose down -v` 会永久删除 MySQL 数据卷。
- Navicat 默认连接 `localhost:3306`，用户 `music_ai`，数据库 `music_ai`，密码以本机 `.env` 为准。

## 常见问题

| 现象 | 优先检查 |
|---|---|
| DeepSeek 调用失败 | `deepseek` Profile 是否启用、Key 是否存在、账户是否可用 |
| 页面能打开但生成无进度 | API 是否运行、SSE 请求是否保持连接、Nginx 是否缓冲响应 |
| MySQL 表未创建 | 数据卷是否已有旧数据、初始化脚本是否只在首次启动执行 |
| MIDI/MusicXML 不可下载 | 任务是否完成、导出目录是否可写、文件元数据是否已保存 |
| 默认启动仍要求模型 Key | 是否误启用了 `deepseek` Profile 或覆盖了默认配置 |

排查时可以记录错误类型、时间和任务 ID，但不得复制密钥、Cookie 或完整认证请求头。
