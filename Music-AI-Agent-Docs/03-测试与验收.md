# 测试与验收

导航：[[00-首页]] · [[AGENTS]] · [[02-开发运行与配置]]

## 测试矩阵

| 范围 | 关键验证 |
|---|---|
| Domain | 分数时值、拍号、调弦和指法 |
| Generator | 提示词差异、可复现性、调性和不同拍号 |
| Export | MIDI 重读、MusicXML 结构和调号 |
| Application | 生成、版本、局部重写和导出物 |
| API/MCP | 接口、鉴权、工具发现和调用 |
| Persistence | Testcontainers MySQL 8 |
| Python | WAV 元数据、响度、BPM 初估和上传校验 |
| LLM配置 | 提供商配置归一化、缺失Key快速失败；真实模型测试默认跳过 |

## 完成标准

- 用户流程可运行，错误有明确响应。
- 关键规则具备自动化测试。
- 不依赖硬编码密钥或个人路径。
- MIDI/MusicXML 经过程序验证。
- Guitar Pro 功能完成人工导入检查。
- [[AGENTS]]、[[PROJECT_STRUCTURE]] 与代码一致。

## 推荐验收顺序

1. 运行 Domain 和 Generator 测试，先确认音乐不变量。
2. 运行 Application、Persistence 和 Export 测试，验证完整纵向链路。
3. 运行 API/MCP 集成测试，检查鉴权、协议和错误响应。
4. 构建 Vue 前端并运行 Python 测试。
5. 使用代表性提示词执行一次真实用户流程。
6. 将 MIDI/MusicXML 导入 Guitar Pro 8，检查速度、拍号、调号、时值与弦品。

## 代表性验收样例

至少覆盖以下输入差异：大调与小调、标准调弦与 Drop D、直拍与切分、低复杂度与高复杂度、常见拍号与非四四拍。每组都应验证“同输入可复现、关键约束变化会改变结果、所有结果继续通过校验”。

> [!check]
> 相同提示词应可复现；不同提示词或关键约束应生成不同事件，同时继续通过时值和可演奏性校验。

MCP验收还应分别覆盖自然语言工具和`generate_guitar_riff_from_constraints`。后者必须在未启用`llm` Profile时完成生成，用来证明外部用户可使用自己的模型而不消耗服务器Key。
